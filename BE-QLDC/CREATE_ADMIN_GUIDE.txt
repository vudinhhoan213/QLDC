========================================
  HƯỚNG DẪN TẠO TÀI KHOẢN ADMIN
========================================

🚀 CÁCH 1: SỬ DỤNG SCRIPT (KHUYÊN DÙNG)
----------------------------------------

1. Mở terminal tại thư mục BE-QLDC

2. Chạy lệnh:
   npm run create-admin

3. Tài khoản admin sẽ được tạo với thông tin:
   Username: admin
   Password: 123456
   Role: TO_TRUONG (Tổ trưởng)


🔧 CÁCH 2: SỬ DỤNG API ENDPOINT
--------------------------------

Nếu muốn tạo bằng API, gọi endpoint sau:

POST http://localhost:3001/users
Headers:
  Content-Type: application/json
  Authorization: Bearer <TOKEN_CỦA_TỔ_TRƯỞNG>

Body:
{
  "username": "admin",
  "password": "123456",
  "role": "TO_TRUONG",
  "fullName": "Quản trị viên",
  "email": "admin@qldc.vn",
  "phone": "0123456789"
}

LƯU Ý: Cách này cần có token của tổ trưởng khác, 
       nên dùng CÁCH 1 để tạo admin đầu tiên.


📝 SAU KHI TẠO ADMIN
---------------------

1. Đăng nhập vào hệ thống:
   - URL: http://localhost:5173/login
   - Username: admin
   - Password: 123456

2. Đổi mật khẩu ngay sau khi đăng nhập lần đầu!

3. Có thể tạo thêm tài khoản khác:
   - Tổ trưởng khác
   - Tài khoản công dân


🔐 BẢO MẬT
-----------

- Mật khẩu được hash bằng bcrypt (10 rounds)
- Token JWT có thời hạn 7 ngày
- Admin có full quyền trong hệ thống


⚠️  LƯU Ý
----------

- Script có thể chạy nhiều lần, sẽ không tạo duplicate
- Nếu admin đã tồn tại, script sẽ hỏi có reset password không
- Nhớ setup file .env với MONGODB_ATLAS và JWT_SECRET


📞 TROUBLESHOOTING
------------------

Lỗi: "Cannot connect to MongoDB"
→ Kiểm tra MONGODB_ATLAS trong .env

Lỗi: "User already exists"
→ Tài khoản admin đã tồn tại, có thể reset password

Lỗi: "Missing required fields"
→ Kiểm tra lại model User có đúng schema không
